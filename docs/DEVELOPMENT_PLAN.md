- **デイリーティーザー**: 翌日シナリオを前日 23:00 に公開するような「ティーザー」運用は未実装。
- **SVGキャラクターイラスト**: Phase 1 はプレースホルダー画像で対応。GDD 想定の専用 SVG は今後の検討対象。

## 2. 今後の開発計画

### 2.1 フェーズと優先度

GDD `docs/gdd/20-schedule.md` のフェーズをベースに、**現状を「Phase 3 コア完了」** として、その先を整理しています。

| フェーズ | 内容 | 優先度 | 依存 |
|----------|------|--------|------|
| **Phase 3 仕上げ** | バグ修正・UI微調整・イラスト差し替え（任意） | 高 | なし |
| **Phase 2 機能（追加モード）** | シリーズモード・ランダムモード | 中 | シナリオテーマ分類・ルーティング設計 |
| **Phase 2 機能（拡散）** | 友達比較・デイリーティーザー | 中 | 結果URL設計の拡張 |
| **Phase 4** | 正式リリース・プレスリリース・配信者アプローチ | 高（リリース時期に応じて） | Phase 3 仕上げ |
| **Phase 5** | 週次シナリオ追加・運用・フィードバック対応 | 継続 | Phase 4 |

### 2.2 Phase 3 仕上げ（推奨: 最優先）

- **目的**: 限定βとしての品質固め。
- **内容**:
  - 既知バグの修正と軽微なUI/UX調整。
  - （任意）シナリオ・タイプのイラストをプレースホルダーから差し替え（`docs/image_prompts/` 等を参照）。
  - Lighthouse・実機での体感速度の確認。
- **成果物**: 安定したβURL、必要に応じて簡易リリースノート。

### 2.3 Phase 2 機能: シリーズモード・ランダムモード

- **目的**: 「毎日1問」に加え、テーマ別・ランダムで複数問をプレイできるようにする（GDD 03 / proposal 04）。
- **主な作業**:
  - シナリオにテーマ/シリーズIDを付与し、シリーズ別リストを取得するデータ設計・`manifest` 拡張。
  - ルート例: `/series/:seriesId`、`/random`（またはクエリでモード指定）。
  - GameContext の初期化を「デイリー」「シリーズ」「ランダム」で切り替え。
  - ランディングから「シリーズを選ぶ」「ランダムで診断」への導線追加。
- **参照**: `.kiro/specs/ai-moral-assessment/design.md` の Non-Goals に「Phase 2」として記載。新規 spec または既存 spec のタスク追加で管理推奨。

### 2.4 Phase 2 機能: 友達比較・ティーザー

- **友達比較**（GDD 18 / proposal 09）  
  - シェアリンク（結果URL）を2つ並べて表示する画面。  
  - 例: `/compare?mine=UEOA&theirs=XXXX` のようなURL設計と、比較用ページの新設。
- **デイリーティーザー**  
  - 翌日シナリオを前日 23:00 に公開する運用は、フロントのみでは完結しない場合あり。  
  - `publishDate` を日付で切るだけであれば現状の manifest で可能。「23:00 公開」を厳密にする場合はバックエンドまたは Workers の軽い API 検討。

### 2.5 Phase 4: 正式リリース

- **目的**: 正式URLでの公開、認知拡大（GDD 20、proposal 07）。
- **内容**:
  - プレスリリース文案・プレスキット整備。
  - 配信者・メディア向け説明（`docs/gdd/18-viral.md` 等を活用）。
  - X（Twitter）等での告知と、必要に応じた免責・報告機能の明文化（GDD 20 リスク対応）。

### 2.6 Phase 5: 継続運用

- **目的**: 週次シナリオ追加・フィードバック対応・KPI確認（GDD 21）。
- **内容**:
  - 新規シナリオ JSON の追加と `manifest.json` の更新。
  - ビルド時シナリオ検証（既存 `validate:scenarios`）を継続利用。
  - アクセス・シェア・OGP 呼び出しなどの簡易計測と、月額コストの監視。
